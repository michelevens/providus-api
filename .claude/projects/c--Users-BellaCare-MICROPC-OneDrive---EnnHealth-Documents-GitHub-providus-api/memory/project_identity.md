The repo "providus-api" is now called **Credentik**. It is a credentialing platform for ALL provider types (NPs, MDs, DOs, PAs, LCSWs, etc.) — not just NPs.

**Why:** The user started with their own NP credentialing but the platform serves all provider types. People are already interested in buying it as a SaaS service.

**How to apply:** Treat this as a product on a path to production. Prioritize production-readiness (security, reliability, data integrity, observability). NP-specific features (collaborative practice, practice authority) should always be optional/conditional. Use "Credentik" as the product name going forward.
